# 1.4 — Calculation Layer

> Part of Phase 1 — Excel System

## Purpose

The calculation layer takes the clean, unified data from `ALL_FINANCIALS` and applies valuation logic across multiple approaches to produce a business enterprise value estimate for the subject company.

## Valuation approaches

| Approach | Primary Sheet | Supporting Sheets | Status |
|---|---|---|---|
| Discounted Cash Flow (DCF) | `DCF` | `WACC`, `NWC` | Active |
| Guideline Public Company (GPC) | `GPC` | `GPC_IS`, `GPC_BS`, `RatioCatalog` | Active |
| Guideline Transaction (GT) | `GT` | — | Active |
| Net Asset Value (NAV) | `NAV` | — | Active |

## How data flows in

All calculation cells reference `ALL_FINANCIALS` using the `Key` column via INDEX/MATCH. The standard formula pattern is:

=IFERROR(INDEX(ALL_FINANCIALS[column], MATCH(key, ALL_FINANCIALS[Key], 0)) * pmlUnits, 0)


Where:
- `key` = `ticker|line item` (lowercase)
- `pmlUnits` = scale factor from the `Control` sheet (currently 1.00 = actuals in millions)

## Key model parameters

All configurable via named ranges on the `Control` and `Dash_Prjctn` sheets. See the Named Range Inventory in `excel-system.md` for the full list.

## Subject company financials

The `IS` and `BS` sheets hold the subject company's historical and projected financial statements. These currently pull from `ALL_FINANCIALS` (public data path). A pending enhancement will add `if(CompanyStatus="Publicly Traded")` two-way binding so that:
- When public → pull from `ALL_FINANCIALS` (stockanalysis data)
- When private → pull from `PBC_IS` / `PBC_BS` (client-provided data)
