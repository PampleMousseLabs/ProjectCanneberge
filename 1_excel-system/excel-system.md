# Phase 1 — Excel System

> Last updated: 2026-07/17
## Overview

The Excel system is the first phase of Project Canneberge. It is a fully functional ETL pipeline and valuation model built inside a macro-enabled Excel workbook (`Project_Canneberge.xlsm`). The system ingests financial data from stockanalysis.com, MarketScreener.com, and fred.stlouisfed.org, transforms it into a clean master lookup table, and feeds a multi-approach business enterprise valuation model.

The system is organized into five sections:

- **1.1 Data Ingestion** — Power Query and VBA fetcher functions that pull raw financial data per ticker
- **1.2 Tier 1 Transforms** — Combiner queries that loop tickers and stack results per statement type
- **1.3 Tier 2 Transform** — Master table (`ALL_FINANCIALS`) combining all statements into one long/tall lookup table
- **1.4 Calculation Layer** — Excel worksheet formulas, valuation approaches, and supporting schedules
- **1.5 Report Output** — Dashboard, projection controls, and valuation summary (Complete/always in progress)

## Section 1.1 — Data Ingestion

Power Query fetcher functions and VBA extraction routines that pull raw financial data per ticker.

### Power Query fetcher functions

Each function takes a ticker string and returns a clean, schema-locked table. stockanalysis functions use `Web.Page()` to render JavaScript-driven tables.

| Function | Source | URL Path | Output |
|---|---|---|---|
| `fnIS` | stockanalysis.com | `/financials/` | Income statement rows |
| `fnBS` | stockanalysis.com | `/financials/balance-sheet/` | Balance sheet rows |
| `fnCFS` | stockanalysis.com | `/financials/cash-flow-statement/` | Cash flow rows |
| `fnRatio` | stockanalysis.com | `/financials/ratios/` | Ratio rows (Market Cap, EV, multiples) |
| `fnBeta` | stockanalysis.com | `/` (overview page) | Single beta row |
| `fnForwardEst` | `tblForwardEst_Raw` (local) | N/A — reads from staging table | Forward estimates per ticker |
| `fnFRED` | fred.stlouisfed.org | REST API per series ID | Single observation per FRED series |

### Utility functions

| Function | Purpose |
|---|---|
| `fnCleanFinancialTable` | Standardizes nulls (`"-"`, `"—"`, `"N/A"`, `""` → null), trims text, normalizes column names |
| `fnSchemaLock` | Enforces column schema contract — adds missing columns as null, removes unexpected columns, reorders consistently |

### VBA extraction (`modExtraction`)

| Stage | Action | Output |
|---|---|---|
| Stage 0 — Slug build | POST to MarketScreener `async/search/quick` per ticker | Slug table (e.g. `ROCKET-LAB-CORPORATION-126208072`) |
| Stage 0.5 — Finance scrape | GET MarketScreener finances page per slug; parse HTML table | `ForwardEst_Raw` staging table |

### Output schema (all fetchers except `fnFRED`)

| Column | Description |
|---|---|
| `Key` | `ticker\|line item` — primary lookup key |
| `Ticker` | Lowercase ticker string |
| `Line Item` | Lowercase metric name |
| `TTM` | Trailing twelve months (IS/BS/CFS) or `Current` (Ratios only) |
| `LFY` through `LFY-3` | Historical fiscal year values (mapped from `FY YYYY` source columns) |
| `NFY` through `NFY+2` | Forward estimate values (`fnForwardEst` only) |

> ⚠️ **Known technical debt:** The column names above represent the *intended* schema after the year-mapping refactor. Currently, year columns are hardcoded as literal year strings (`"2025"`, `"2024"`, etc.) in several M files.

## Section 1.3 — Tier 2 Transform

`ALL_FINANCIALS` is the master ETL output and primary source of truth for the calculation layer. It calls all fetcher functions **directly** (not via Tier 1 queries — this avoids async race conditions), wraps each result in `Table.Buffer`, combines all results into one long/tall table, and applies a final column select and reorder.

**Output columns:** `Key`, `Ticker`, `Line Item`, `TTM`, `Current`, and year columns (currently hardcoded; will be dynamic after year-mapping refactor)

**Lookup pattern:** All calculation layer formulas use `Key` (`ticker|line item`) as the INDEX/MATCH lookup value against this table.

## Known Limitations

- `Web.Page()` renders a full browser engine per fetch — 6 functions × N tickers = significant runtime (~24 min for 10 tickers)
- Yahoo Finance `v10/quoteSummary` endpoint now requires crumb authentication — not usable unauthenticated; `pmlPRICE` uses `v8/finance/chart/` instead
- `"Pro"` paywalled values on stockanalysis.com are automatically nulled out
- VBA async pipeline means `ALL_FINANCIALS` cannot safely reference Tier 1 query outputs (race condition risk) — it calls fetcher functions directly instead
- MarketScreener has a daily pull limit — exceeding it returns a ~284KB generic page; auto-dumped to `%TEMP%\Canneberge_FailedFinance\` for diagnosis
- `ETL_LOG` auto-clears on each pipeline run — no historical log retention
