# 1.3 — Transformation — Tier 2

> Part of Phase 1 — Excel System

## Purpose

Tier 2 produces the single master lookup table (`ALL_FINANCIALS`) that the entire calculation layer depends on. It combines all financial statement types, forward estimates, and ratios into one long/tall table indexed by `Key` (`ticker|line item`).

## How it works

`ALL_FINANCIALS` calls all fetcher functions **directly** — it does not reference the Tier 1 output sheets. This is intentional: since Power Query refreshes are async, referencing Tier 1 outputs would create race conditions where `ALL_FINANCIALS` might read stale or mid-refresh data.

Each fetcher result is wrapped in `Table.Buffer` to prevent redundant re-evaluation, then all results are combined via `Table.Combine` and a final column select/reorder is applied.

## Output schema

| Column | Description |
|---|---|
| `Key` | `ticker\|line item` (lowercase) — primary lookup column |
| `Ticker` | Lowercase ticker string |
| `Line Item` | Lowercase metric name |
| `TTM` | Trailing twelve months value |
| `Current` | Current/spot value (ratios) |
| Year columns | Historical (LFY through LFY-3) and forward (NFY through NFY+2) |

## Lookup pattern

All calculation layer formulas use:

=IFERROR(INDEX(ALL_FINANCIALS[column], MATCH(key, ALL_FINANCIALS[Key], 0)) * pmlUnits, 0)
