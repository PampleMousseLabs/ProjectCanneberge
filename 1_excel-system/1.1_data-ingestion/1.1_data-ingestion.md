# 1.1 — Data Ingestion

> Part of Phase 1 — Excel System

## Purpose

Data ingestion is the entry point of the ETL pipeline. It pulls raw financial data from external sources on a per-ticker basis and returns clean, schema-locked tables ready for combination in Tier 1 transforms.

## How it works

There are two ingestion pathways:

### Pathway 1 — Power Query fetcher functions (stockanalysis.com + FRED)

Six fetcher functions hit stockanalysis.com via `Web.Page()` (which renders the JavaScript-driven tables in a headless browser context). Each takes a ticker string and returns a table.

| Function | URL Path | Content |
|---|---|---|
| `fnIS` | `/financials/` | Income statement |
| `fnBS` | `/financials/balance-sheet/` | Balance sheet |
| `fnCFS` | `/financials/cash-flow-statement/` | Cash flow statement |
| `fnRatio` | `/financials/ratios/` | Ratios and multiples |
| `fnBeta` | `/` (overview page) | Beta value |
| `fnFRED` | `api.stlouisfed.org` REST | Interest rate per FRED series ID |

`fnForwardEst` is technically also a fetcher function but reads from a local staging table (`tblForwardEst_Raw`) rather than the web.

All stockanalysis fetchers pass through two utility functions:
- **`fnCleanFinancialTable`** — null standardization, text trimming
- **`fnSchemaLock`** — column enforcement (add missing, remove extra, reorder)

### Pathway 2 — VBA HTTP extraction (MarketScreener)

`modExtraction` performs two synchronous stages before Power Query runs:

1. **Stage 0 — Slug Build:** POST to MarketScreener search API to resolve each ticker to an internal company slug
2. **Stage 0.5 — Finance Scrape:** GET the finances page for each slug, parse the HTML table for forward consensus estimates (Net Sales, EBITDA, EBIT, Net Income), write to `ForwardEst_Raw` staging table

## Source column formats

| Source | First Column | Year Columns | Format |
|---|---|---|---|
| stockanalysis — IS, BS, CFS | `TTM` | `FY 2025`, `FY 2024`, ... | `FY ` prefix |
| stockanalysis — Ratios | `Current` | `FY 2025`, `FY 2024`, ... | `FY ` prefix |
| MarketScreener | — | `2023`, `2024`, ..., `2028` | Raw `YYYY` |
